# Лог шага 2

Связанный этап: `Этап 1. Выбор Инструмента И Контракта`

Статус: завершён

Цель шага:

- выбрать первый прикладной инструмент;
- зафиксировать источник данных, входной контракт, ожидаемый результат и пользовательскую команду.

Выполненные действия:

1. В отдельном документе `docs/mcp-tool-integration-contract.md` зафиксирован первый vertical slice для интеграции инструмента.
2. В качестве первого источника данных выбран публичный mock API `JSONPlaceholder`.
3. В качестве первого MCP-инструмента выбран `fetch_post`.
4. Зафиксирован обязательный входной параметр `postId`.
5. Зафиксирована пользовательская команда `tool post <postId>`.

Принятые решения:

- начать с HTTP mock API, а не с Git или другой интеграции, чтобы минимизировать инфраструктурную сложность;
- использовать сценарий чтения публикации как самый простой и понятный пример реального `tools/call`;
- в первом проходе ориентироваться на читаемый CLI-результат, даже если позже появится более структурированная проектная модель ответа.

Проверка:

- контракт первого инструмента теперь явно зафиксирован в отдельном документе, на который можно опираться на серверном, агентном и presentation-этапах.
