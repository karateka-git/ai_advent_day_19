## Дополнение. Direct Launcher-Запуск Без Gradle-Шума

Статус: завершён

Цель:

- убрать из ручного запуска клиента и сервера шум Gradle вроде `> Task`, daemon-сообщений и progress bar;
- перенести ручной и scripted-запуск на прямые `.bat`-артефакты по образцу `day_15`;
- сохранить корректную UTF-8-кодировку для русскоязычного вывода и scripted-ввода.

Выполненные действия:

1. В `build.gradle.kts` добавлены отдельные distribution-task:
   - `installClientDist`
   - `installServerDist`
2. Для клиента и сервера добавлены прямые launcher-артефакты:
   - `build\install\mcp-client\bin\mcp-client.bat`
   - `build\install\mcp-server\bin\mcp-server.bat`
3. `start-manual-check.ps1` переведён на запуск этих `.bat` вместо `gradlew runServer/runClient`.
4. `check-e2e.ps1` переведён на прямой server launcher и scripted client launcher поверх клиентского `.bat`.
5. `invoke-client-commands.ps1` адаптирован под прямой клиентский `.bat` и запись stdin как UTF-8 без BOM через `BaseStream`.
6. README и локальный `MemoryBank` синхронизированы с новым способом запуска.

Принятые решения:

- direct launcher считать основным способом ручного запуска на Windows;
- Gradle-задачи `runServer` и `runClient` оставить как технические entrypoint-ы, но не как предпочтительный UX ручной проверки;
- для scripted-клиента сохранить паттерн из `day_15`: `cmd.exe`, `chcp 65001`, `JAVA_TOOL_OPTIONS` и явный UTF-8 для stdin/stdout/stderr.

Проверка:

- `.\gradlew.bat test` завершается успешно;
- `.\gradlew.bat installClientDist installServerDist` завершается успешно;
- `powershell -ExecutionPolicy Bypass -File .\scripts\check-e2e.ps1` завершается успешно;
- `powershell -ExecutionPolicy Bypass -File .\scripts\start-manual-check.ps1 -Headless -SkipBuild` завершается успешно;
- headless-вывод подтверждает, что в клиентской консоли больше нет Gradle progress UI, а остаётся только вывод самого приложения и MCP-логов.
